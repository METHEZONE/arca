# Agent notes for this repository

**Read this before classifying the project.** This repo contains TWO separate
products. Do not assume the whole repo is a web app because `package.json`
sits at the root, and do not call anything here Electron — nothing in this
repo uses Electron.

## 1. `apps/arca/` — ARCA companion (native Apple apps)

- Swift 6 / SwiftUI. Targets: `ARCA` (iOS + macOS), `ArcaVoiceWatch`
  (watchOS), `ArcaVoiceShare` (share extension), `ArcaVoiceWidgets`
  (widgets / Live Activity), `ARCA-Test` (isolated macOS test app).
- Project is generated: edit `apps/arca/project.yml`, then run
  `xcodegen generate` (the committed `ARCA.xcodeproj` is the output).
- Build check: `xcodebuild -project ARCA.xcodeproj -scheme ARCA
  -destination "generic/platform=iOS" build CODE_SIGNING_ALLOWED=NO`
- Ships to TestFlight as `com.thezone.arca.voice`. Build numbers are
  auto-managed at export time (`manageAppVersionAndBuildNumber`) — do not
  bump `CFBundleVersion` by hand.
- Shared engine lives in `apps/arca/Packages/ArcaVoiceKit`.
- Info.plist files are REGENERATED by xcodegen from `project.yml`
  properties — put plist keys in `project.yml`, not in the plists.

## 2. Root — ARCA web dashboard (Next.js)

- Next.js + TypeScript. `npm run dev` (port 4174), `npm run build`,
  `npm run lint`.
- Deployed on Vercel. Demo mode works with zero API keys.

## Cross-cutting

- The iPhone/Mac apps converge through a GitHub-repo relay
  (`RelaySync.swift`) — tasks and session transcripts sync; audio stays
  local per device.
- Secrets never enter git: `BundledKeys.plist`, `~/.arca/*`, and
  App Store Connect keys are all untracked. Keep it that way.
