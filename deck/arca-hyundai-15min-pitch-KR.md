  1. 대화 요약
  2. 해야 할 일(태스크)

  4. 이메일/Slack 답장 초안
  5. 추적용 기록 항목(팀이면 CRM entry)

- ARCA가 하는 일 = 받아쓰기보다 한 단계 뒤:
  - 누가 뭘 해야 하는지 **판단** → 캘린더/메일 **초안** 생성 → 저위험 기록은 **즉시 저장** → 사람이 결정할 발송은 **승인 카드**로 올림.
- 발표자가 누르는 것은 하나 — **"arca it".** 여기서 **루프가 닫힘.**
- 메시지: AI가 말 잘하는 게 아니라, **내가 캘린더 열고 메일 쓰고 회의록 복사 안 해도 일이 다음 단계로 넘어갔다**는 것.
- ⚠️ 무대 리스크 대비: Action Pack 5요소가 보이는 **90초 폴백 영상** 필수 준비.

## 5 — 신뢰 모델: AI가 잘못 보내면? (7:45–8:30)

  - **저위험 → 자동.** 내부 확인, 합의된 일정 후보 전달, 회의록 저장.
  - **중위험 → 초안+승인.** 외부 파트너 메일, 고객 후속, 돈 걸린 문서 공유.
  - **고위험 → 무조건 사람.** 계약, 가격 확정, 법무, HR, 갈등, 낯선 외부 이해관계자.
- 모든 자동 발신에 **"ARCA-assisted" 표시** + 전 행동 **audit log.**
- 신뢰는 "AI가 안 틀린다"가 아니라 **어디까지 맡겼는지 보이고, 되돌릴 수 있고, 위험한 건 승인 없이 안 나가는 구조**에서 옴.

# 부록 A — Q&A 핵심 답변 (음슴체)

- **왜 박민성인가** — 이 문제의 사용자 0번. 제품·데모·리서치·배포 직접 실행. solo 리스크는 인정 → 그래서 박진성·하승호와 팀 구성, 게임 개발자 영입이 다음 과제.
- **왜 지금인가** — 2년 전엔 화자분리 STT + LLM 분석 + 도구 실행을 싸게 묶기 어려웠음. 지금은 STT·LLM이 충분히 싸고, 사용자도 AI에 초안·실행 맡기는 데 익숙. 기록 앱 다음 카테고리(완수/위임)가 열림.
- **누가 돈 내나** — 초기 개인 지식노동자(Pro 12,900). 팀은 좌석 19,900. 단 현재 가격은 가설, 90일 핵심 지표가 유료 의사·반복 사용.
- **MS·Notion·Granola가 하면 끝 아닌가** — 모델은 해자 아님 인정. 그러나 요약 앱은 기록에서 멈추고, suite는 자기 생태계에 갇힘. 우리는 사용자가 이미 쓰는 도구를 가로질러 후속조치를 닫음. 장기 해자 = 완수 데이터·신뢰 인터페이스·동사 습관 + 캡처 하드웨어.
- **Bond이 먼저 했잖아** — 같은 문제를 검증한 건 시장 증거. 그러나 Bond은 소프트웨어-온리 + 미니멀 리스트. 우리는 캡처 하드웨어 + 컴패니언 delight로 걔네가 비운 레인을 먹음. 걔네 CEO가 직접 적은 약점(캡처 안 됨)을 우리가 친다.
- **AI 잘못 보내면** — 저위험만 자동, 중위험 초안+승인, 고위험 사람. 전 발신 "ARCA-assisted" 표시 + audit log.
- **하드웨어 꼭 필요한가** — v1 PMF는 소프트웨어로 검증. 하드웨어는 소프트웨어가 구조적으로 못 푸는 "캡처 레이어"를 먹는 해자 + 이 팀이라 가능한 차별점.
- **제일 큰 리스크** — ① 진짜 돈 내는지(WTP) ② AI 실행 신뢰 ③ 안 넓어지고 한 루프에 집중. 90일 목표가 정확히 이걸 죽이거나 살림.
